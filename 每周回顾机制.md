# 每周回顾机制

## 🎯 目的
建立系统化的学习和协作回顾机制，确保：
1. **巩固记忆**：重要内容不会被遗忘
2. **持续优化**：发现问题并及时改进
3. **共同成长**：跟踪学习进度和成果
4. **关系深化**：加强协作默契和信任

## 📅 时间安排

### 每周回顾时间
- **频率**：每周一次
- **建议时间**：周日晚上 20:00-21:00（北京时间）
- **时长**：约30-60分钟
- **灵活性**：可根据实际情况调整

### 回顾触发
1. **自动提醒**：通过cron设置每周提醒
2. **手动触发**：随时可以启动回顾
3. **重要事件后**：完成重要项目或学习后

## 📋 回顾内容框架

### 1. 学习成果回顾
```
✅ 本周学习了什么？
- 新知识/技能
- 工具使用方法
- 概念理解
```

### 2. 项目进展回顾
```
🚀 本周完成了什么？
- 项目里程碑
- 问题解决
- 技术突破
```

### 3. 协作优化回顾
```
🤝 协作体验如何？
- 沟通效率
- 工具使用体验
- 流程优化点
```

### 4. 问题与挑战
```
⚠️ 遇到了什么困难？
- 技术问题
- 理解障碍
- 时间/资源限制
```

### 5. 下周计划
```
📅 下周要做什么？
- 学习目标
- 项目计划
- 待解决问题
```

### 6. 想法分享
```
💭 本周的思考和洞察
- MOSS的观察和建议
- 飞天的想法和感受
- 共同发现的规律
```

## 🔧 实施步骤

### 第一步：设置提醒机制
由于cron配置需要特定格式，我们将采用以下提醒方式：

1. **手动提醒**：我会在每周日20:00主动发起回顾
2. **文件提醒**：创建`memory/本周回顾提醒.md`文件
3. **对话提醒**：通过日常对话提醒回顾时间

**临时方案**：我会在每周日20:00左右主动发起回顾对话。

### 第二步：创建回顾模板文件
在每周回顾时，我会自动创建一个新的回顾文件：
```
memory/回顾-YYYY-MM-DD.md
```

### 第三步：回顾流程
1. **自动触发**：收到提醒后开始回顾
2. **收集数据**：读取本周的记忆文件和对话记录
3. **生成报告**：基于框架生成初步回顾报告
4. **共同讨论**：我们一起补充和完善
5. **保存记录**：保存到记忆文件并更新MEMORY.md

### 第四步：行动项跟踪
- 将重要的行动项添加到TODO列表
- 设置下周的检查点
- 更新学习计划和项目路线图

## 🔄 持续改进

### 机制优化
- 每月评估回顾机制的效果
- 根据反馈调整回顾内容和频率
- 优化自动化程度和用户体验

### 价值最大化
- 确保回顾内容对实际学习和协作有帮助
- 避免形式主义，注重实质内容
- 将回顾成果转化为实际行动

## 💡 为什么这个机制重要

### 对学习
- **防止遗忘**：定期巩固学习成果
- **系统化学习**：建立学习节奏和纪律
- **持续进步**：明确下一步学习方向

### 对协作
- **默契培养**：通过定期交流加深理解
- **问题预防**：及时发现和解决协作问题
- **效率提升**：优化工作流程和沟通方式

### 对关系
- **信任建立**：通过系统化合作建立可靠关系
- **共同成长**：见证彼此的进步和变化
- **长期合作**：为持续协作奠定基础
