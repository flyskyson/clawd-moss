# 多角色Clawdbot快速启动指南

## 🚀 一键启动全部角色

```bash
# 启动所有角色
./scripts/manage-roles.sh start

# 查看运行状态
./scripts/manage-roles.sh status

# 查看实时日志
./scripts/manage-roles.sh logs
```

## 📋 单个角色启动

### 启动领航者（🚀）
```bash
./scripts/start-leader.sh
```

### 启动哲思者（💡）
```bash
./scripts/start-thinker.sh
```

### 启动实干家（⚡）
```bash
./scripts/start-executor.sh
```

### 启动和谐者（🤝）
```bash
./scripts/start-coordinator.sh
```

## 🔄 日常操作

### 每日组长轮换
```bash
# 手动触发轮换（系统会在每天0:00自动执行）
./scripts/组长轮换系统.sh

# 查看今日议题
cat state/today-topic.md
```

### 任务处理
```bash
# 检查新任务
./scripts/任务协作系统.sh check

# 更新任务进度
./scripts/任务协作系统.sh update <任务ID> <进度百分比> "<更新说明>"

# 标记任务完成
./scripts/任务协作系统.sh complete <任务ID>

# 生成任务报告
./scripts/任务协作系统.sh report

# 查看任务状态
./scripts/任务协作系统.sh status
```

### 角色管理
```bash
# 停止所有角色
./scripts/manage-roles.sh stop

# 重启所有角色
./scripts/manage-roles.sh restart

# 查看运行状态
./scripts/manage-roles.sh status
```

## 📝 创建新任务

在 `tasks/` 目录下创建任务文件：

```markdown
# 任务标题

## 任务描述
详细描述任务内容和目标

## 优先级
高/中/低

## 截止时间
2026-02-02 18:00

## 具体要求
1. 要求1
2. 要求2
3. 要求3
```

文件命名格式：`task-<编号>.md` 或 `task-<编号>.txt`

## 🔍 故障排查

### 角色无法启动
```bash
# 检查配置文件
cat config/roles/leader-config.json

# 检查端口是否被占用
lsof -i :18800

# 查看错误日志
tail -f logs/*.log
```

### 角色无法加入群组
1. 检查飞书配置中的群组ID是否正确
2. 确认群组权限设置
3. 检查Clawdbot的飞书连接状态

### 脚本执行失败
```bash
# 检查脚本权限
ls -l scripts/*.sh

# 重新设置权限
chmod +x scripts/*.sh

# 检查jq工具
which jq
```

## 💡 使用技巧

1. **循序渐进**：建议先启动一个角色测试，确认正常后再启动全部
2. **定期备份**：定期备份workspaces/和state/目录
3. **日志监控**：使用`./scripts/manage-roles.sh logs`实时监控
4. **定期检查**：每天检查角色状态和任务进度

## 🛠️ 配置调整

### 修改轮换时间
编辑 `config/feishu-group-config.json`：
```json
"rotation_schedule": "09:00"
```

### 调整议题模板
编辑 `scripts/组长轮换系统.sh` 中的 `TOPIC_TEMPLATES` 数组

### 自定义角色性格
编辑 `roles/*-SOUL.md` 文件
